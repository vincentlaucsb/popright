# Architecture Decision Records

Popright keeps ADRs split by package layer:

- `core/` records framework-neutral decisions about menu data, DOM behavior, positioning, accessibility, lifecycle, and theming.
- `react/` records React-specific decisions about hooks, composition, ownership, prop forwarding, and migration ergonomics.

ADRs should be short, dated, and practical. They should capture decisions that would be easy to accidentally undo later.
